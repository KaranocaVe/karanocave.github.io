《C语言解惑》批注

在此记录一些迷惑性较强的谜题。

第一章 操作符

1.1.3

int x; x = - 3 * 4 % - 6 / 5 ;

* % / 同级，从左到右关联。

1.3.4

z = x +++++ y; //因为遵循原则“构成操作符的字符个数越多越好”，所以会被解释为 z = x ++ ++ + y; //而非 z = x ++ + ++ y;

1.3.5

int x,z; z = x / ++ x ;

顺序未定义。
